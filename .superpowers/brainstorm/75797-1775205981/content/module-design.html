Internal Steps

1
Load prompt template
Read prompts/briefing.md from disk relative to project root
2
Build user message
Compose Today is {date}.\n\n{registry JSON}
3
Call Claude API
Direct fetch to api.anthropic.com/v1/messages — no SDK dependency needed since Bun has native fetch
4
Extract text response
Pull content[0].text from API response, return as-is

Decisions

Direct fetch, no SDK
The Anthropic SDK is nice but adds a dependency. The API is a single POST — raw fetch keeps things zero-dependency and consistent with how telegram.ts already works.
API key from env
ANTHROPIC_API_KEY env var — same pattern as TELEGRAM_BOT_TOKEN. Config only holds the model name.
No retry logic
If the API fails, throw. The CLI already logs errors. Don't add complexity for a daily cron that can just run again.
max_tokens: 1024
Keeps output concise and cost negligible. Telegram limit is 4096 chars — 1024 tokens ≈ ~800 chars, well within bounds.
